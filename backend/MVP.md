# 🗓️ SlowMe MVP – 주차별 개발 목표

> 핵심: **기술 데모 ❌ / 완성된 경험 1개 ✅**  
> 철학: **침묵이 허용되는 놀이 공간**, 기능 추가보다 **제거에 집중**

---

## 🟩 1주차: 서버 뼈대 + 표준 고정 주간

### 목표

“이 프로젝트는 이런 구조로 간다”를 **코드로 고정**

### 핵심 테마

- 인프라 / 표준 / 구조
- 기능 욕심 ❌

### ✅ 반드시 완료 (Backend / Spring Boot)

#### 프로젝트 기본 구조(패키지 고정)

`com.unocode.slowme`

- `user`
- `room`
- `common`
- `config`

#### 환경 분리

- `local` / `test` profile
- DB 연결 성공 (H2 또는 MySQL)

#### 로깅 표준

- Log4j2 + AsyncLogger
- profile 별 `log4j2` 분리
- requestId (MDC 기반) 준비

#### 기본 도메인 설계(초안)

- User: `(id, nickname)`
- Room: `(id, type, hostId, maxParticipants)`
- Participant: `(userId, roomId, status)`

#### 헬스 체크

- `GET /health` (서버 정상 기동 확인)

### ❌ 하지 않음

- 인증
- UI 연동
- 소리 처리
- 실시간 통신

### 🧪 완료 기준(DoD)

- 서버 실행 시 에러 ❌
- profile 변경해도 로그 정상
- `/health` 호출 성공
- DB 테이블 자동 생성

## 🟦 2주차: 같이 놀기 – 방 흐름 완성

### 목표

말 없는 “같이 있기”의 서버 플로우 완성

### 핵심 테마

- Room Lifecycle
- 상태 관리

### ✅ 반드시 완료 (Backend)

#### 방 생성

- `POST /rooms`
- RoomType: `TYPING` / `DRAW`

#### 방 목록 조회

- `GET /rooms`
- 현재 인원 수 포함

#### 방 입장/퇴장

- `POST /rooms/{roomId}/join`
- `POST /rooms/{roomId}/leave`

#### 방 종료 조건

- host가 나가면 방 종료
- 마지막 인원이 나가면 방 종료

#### 이벤트 로그(최소)

- `ROOM_CREATED`
- `USER_JOINED`
- `USER_LEFT`

### ❌ 하지 않음

- WebSocket
- 실시간 소리 동기화
- 채팅
- 알림

### 🧪 완료 기준(DoD)

- 방 생성 → 입장 → 퇴장 플로우가 끊김 없이 동작
- 잘못된 상태 요청 시 에러 반환
- 로그로 방 상태 추적 가능

## 🟨 3주차: 혼자 놀기 + 클라이언트 연결

### 목표

앱에서 “쓸 수 있는 느낌”을 처음 만든다

### 핵심 테마

- UX 시작
- 단방향 경험

### ✅ 반드시 완료

#### Frontend (React Native)

- 첫 화면: 혼자 놀기 / 같이 놀기 선택
- 혼자 놀기:
  - ASMR 타입 선택
  - 로컬 사운드 재생
- 같이 놀기:
  - 방 리스트 조회
  - 방 생성
  - 방 입장

#### Backend

- 방 API 프론트 연동
- CORS 설정

### ❌ 하지 않음

- 실시간 반응
- 소리 동기화
- 팔로우

### 🧪 완료 기준(DoD)

- 앱 켜서 혼자 놀기 가능
- 방 생성 후 다른 기기에서 입장 가능
- 크래시 ❌

## 🟥 4주차: 마무리 + MVP 완성

### 목표

“누군가에게 보여줄 수 있는 상태” 만들기

### 핵심 테마

- 안정성
- 사용성

### ✅ 반드시 완료

#### 팔로우 기능(최소)

- 팔로우 / 언팔로우
- 팔로우 수 조회

#### 방 UX 개선

- 현재 참여 인원 표시
- 방 상태 실시간 갱신 (polling)

#### 품질/운영 표준 정리

- 예외 처리 정리:
  - 공통 에러 응답
  - 잘못된 요청 처리
- 로그 정리:
  - 주요 이벤트 로그 정리
  - 운영 기준 로그 레벨 조정

### ❌ 하지 않음

- 추천 알고리즘
- 결제
- 통계
- AI

### 🧪 완료 기준(DoD)

- 처음 온 사람도 사용 가능
- 설명 없이도 흐름 이해 가능
- 데모 시 막힘 없음
